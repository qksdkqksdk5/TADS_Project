🚀 Team Collaboration Guide

효율적인 협업과 안전한 배포를 위한 Git 워크플로우

Upstream (원본) ↔ Origin (내 깃허브) ↔ Local (내 컴퓨터)

1. 최초 환경 설정 (딱 한 번만)

프로젝트 참여 시 본인 계정으로 Fork 후 아래 명령어를 순서대로 입력하세요.

# 1. 내 깃허브에서 클론 받기
git clone [내_Origin_주소]

# 2. 팀장 원본 저장소를 Upstream으로 등록
git remote add upstream [팀장_Upstream_주소]

# 3. 연결 확인 (origin, upstream 둘 다 떠야 함)
git remote -v

2. 매일 작업 루틴 (작업 시작 전 필수)

⚠️ 주의: 절대로 main 브랜치에서 직접 코딩하지 마세요!

# 1. 내 로컬 메인을 최신화
git checkout main
git pull upstream main

# 2. 새로운 기능을 위한 전용 브랜치 생성
git checkout -b feature/기능명

3. 결과물 제출하기 (Push & PR)

작업이 완료되면 내 깃허브에 올리고 팀장에게 검토를 요청합니다.

# 1. 변경사항 저장
git add .
git commit -m "feat: 로그인 기능 구현 완료"

# 2. 내 깃허브(Origin)에 브랜치 푸시
git push origin feature/기능명

➡️ GitHub 페이지 접속: "Compare & pull request" 버튼 클릭하여 PR 생성

4. 협업 골든 룰

구분 지켜야 할 규칙
Clean Main 로컬의 main 브랜치는 항상 Upstream과 동일한 청정 상태 유지
PR 전 Pull PR을 날리기 직전에 반드시 upstream main을 한 번 더 pull 할 것
브랜치 네이밍 feature/기능, fix/버그수정 등 목적이 명확한 이름 사용
공통 파일 설정 파일(docker-compose, .env 등) 수정 시 반드시 팀장과 사전 협의

5. 코드가 꼬였다면? (SOS)

작업 중인 브랜치가 엉망이 됐다면 당황하지 말고 브랜치를 새로 만드세요.

git checkout main
git branch -D 망가진_브랜치_이름
git pull upstream main
git checkout -b 새_브랜치_이름
